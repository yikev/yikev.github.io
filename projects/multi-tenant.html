Multi-Tenant File Sharing Platform

Secure tenant-scoped storage, role based access, and shareable links with audit trails.

  • GitHub
  • Back to Projects

Overview

A SaaS-style file sharing app that isolates organizations by tenant while keeping costs low with a shared storage account. Users can upload, preview, and share files with fine grained permissions. Admins track usage and activity from a clean dashboard.

Highlights

  • JWT authentication with refresh and role based authorization.
  • Tenant scoped storage using Cloudinary folders - tenant_id/project_id/filename.
  • Share links with expirations and optional passwords.
  • Audit logging for uploads, downloads, deletes, and permission changes.
  • Dashboard tiles: total files, storage used, active users, recent uploads.
  • Search and filters by filename, type, uploader, project, and date.
  • Size caps and MIME whitelist on the server to prevent unsafe uploads.

Tech Stack

  • Frontend - React, Mantine UI.
  • Backend - FastAPI, Python, SQLAlchemy.
  • Database - Postgres (Supabase in deployment).
  • Storage - Cloudinary shared account with tenant specific paths.
  • Infra - Render.com for API, GitHub Actions for CI.

Architecture

The client authenticates with the FastAPI service and receives JWTs. For uploads, the client requests a short lived Cloudinary signature from the API, uploads the file directly to Cloudinary under the tenant folder, and then posts file metadata back to the API. Downloads use signed URLs and authorization checks to enforce tenant and role boundaries.

Key Endpoints

  • POST /auth/login and /auth/refresh - token based auth.
  • POST /files/sign - returns upload signature and path.
  • POST /files - persist metadata after upload.
  • GET /files?filters=... - search and pagination.
  • GET /files/{id}/download - authorized signed read URL.

Data Model

  • Tenant, Project, User with role membership.
  • File - name, path, size, type, checksum, uploader, project.
  • AuditLog - actor, action, target, timestamp, ip.

Tenancy and Security

  • Authorization checks include tenant id and role on every request.
  • Server side validation for file size and MIME type.
  • All storage paths are prefixed by tenant, preventing cross tenant access.
  • Optional link passwords and expirations for external sharing.
