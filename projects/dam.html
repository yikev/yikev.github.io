Digital Asset Management (DAM) System

Enterprise grade DAM for engineers and project teams - secure uploads, rich metadata, fast search, and governance.

  • Back to Projects

Overview

I built a secure Digital Asset Management platform to help teams store, organize, and find project files at scale. The system supports image and document uploads with automatic metadata extraction, role based access control, tenant scoped permissions, audit logging, and fast search across tags, EXIF, and custom fields.

Highlights

  • ASP.NET Core API with Entity Framework Core and SQL Server.
  • Azure Blob Storage for durable object storage with SAS tokens.
  • Next.js and React UI with server rendered pages and API routes.
  • Role based access - Admin, Manager, Contributor, Viewer.
  • Audit logs for uploads, edits, downloads, and permissions changes.
  • Advanced search - filename, tag, EXIF, GPS, date, and project filters.
  • Admin dashboards for storage usage, recent uploads, and active users.

Tech Stack

  • Backend - ASP.NET Core 8, EF Core, SQL Server, REST, JWT auth.
  • Storage - Azure Blob Storage, ImageKit for thumbnails.
  • Frontend - Next.js 14, React, Mantine UI.
  • Infra - Azure App Service, CI scripts, environment secrets.
  • Utilities - OpenCage Geocoding for city and country from GPS.

Architecture

The API validates metadata and writes asset objects to Azure Blob Storage. Database rows hold normalized file metadata, tags, and audit events. Thumbnails are generated on upload and served through a CDN friendly path. The UI calls the API with JWTs and enforces per tenant and per role access rules.

Key Entities

  • Files - file name, path, size, owner, project, checksum.
  • FileMetadata - GPS coordinates, date taken, camera, tags.
  • Tenants - org scoped isolation and quotas.
  • Projects - group files by client or job.
  • AuditLog - who did what and when.

Notable Endpoints

  • POST /api/files/upload - multi part upload, metadata validation, SAS write.
  • POST /api/files/metadata - save or update metadata after upload.
  • GET /api/files/search - full filter query with pagination and sort.
  • GET /api/files/{id} - signed read for secure download.

Security and Governance

  • JWT based auth with role checks on every route.
  • Per tenant isolation in storage paths - tenant_id/project_id/filename.
  • Server side input validation and allowed MIME checks.
  • Least privilege SAS tokens with short expirations.
  • Structured audit logs for access review and incident response.

What I Built

  • Designed database schema and EF Core models for Files, FileMetadata, Projects, and AuditLog.
  • Implemented upload flow with validation and thumbnail generation.
  • Built search API and React UI for filters, sorting, and pagination.
  • Added admin dashboards for usage stats and recent activity.
  • Integrated OpenCage to backfill city and country from GPS when missing.
