Basketball Fantasy Predictor

A web app that predicts NBA fantasy points for a player using machine learning, combining recent performance, season stats, and game context into a single pre-game projection.

  • Back to Projects

Status: Not deployed yet. Repo includes training notebooks, a FastAPI inference service, and a React UI to run locally.

Overview

A machine learning web app that predicts NBA fantasy points for a searched player before an upcoming game. The UI surfaces season averages, recent form (last 3 games), and key context signals used by the model, then outputs a predicted fantasy points estimate. Built to make player projection exploration fast and explainable.

Highlights

  • Player search with a focused single-player results page.
  • General DraftKings-style fantasy points target (approximate scoring).
  • Feature engineering recomputed at request time to match training-time features.
  • XGBoost regression model for tabular prediction.
  • Recent games panel to compare actual fantasy points vs model expectations.
  • Clear API handling for missing players, missing games, and insufficient history.

Tech Stack

  • Frontend - React, TypeScript, plain CSS.
  • Backend - FastAPI, Python.
  • ML - XGBoost, pandas, scikit-learn (preprocessing + evaluation).
  • Data - Kaggle NBA game logs / stats dataset.
  • Tooling - Jupyter notebooks for training and experiments.

Architecture

The React client calls the FastAPI service to search players and fetch prediction context. On prediction, the API loads the playerâ€™s historical game logs from the Kaggle dataset, recomputes engineered features (for example last-3/5 averages, season aggregates, volatility, and simple game context), validates the feature vector, and feeds it into a trained XGBoost model. The API returns the predicted fantasy points along with key inputs used, so the UI can show both the result and supporting context.

Key Endpoints

  • GET /players/search?q=... - player lookup by name.
  • GET /players/{player_id}/summary - season averages, last-3 averages, and recent games.
  • POST /predict - predicted fantasy points plus model feature inputs.
  • GET /player_history?player_id=... - cleaned game history used for features (optional).

Data Model

  • Player - id, name, team, position.
  • GameLog - date, opponent, home/away, minutes, box score stats, fantasy_points.
  • Features - engineered inputs (recent averages, season stats, volatility, context flags).
  • Prediction - player_id, predicted_points, model_version, generated_at.

Fantasy Points Target

The model predicts a general fantasy points metric based on a DraftKings-style scoring approach. The target is computed from box score stats (for example points, rebounds, assists, steals, blocks, and turnovers), producing a single fantasy points value per game used for training and evaluation.

Model and Features

  • Model: XGBoost regression.
  • Recent form: last 3 and last 5 game averages (and trends where applicable).
  • Season baseline: season-to-date averages.
  • Stability: rolling standard deviation / volatility features.
  • Context: home/away and schedule flags (for example back-to-back), when available in the dataset.
