Eternal Gauntlet - 2D Turn-Based Game

Custom C++ game with an ECS architecture, adaptive AI combat, and procedurally generated maps.

  • GitHub
  • Back to Projects

Overview

Eternal Gauntlet is a lightweight, performance minded 2D turn based game. I built a bespoke Entity Component System to manage thousands of entities efficiently, then layered in a turn queue, status effects, and AI that adapts to player moves. Levels are generated at runtime with seeded randomness so runs are replayable for debugging and balancing.

Highlights

  • Custom ECS with contiguous storage and minimal heap churn.
  • Deterministic turn system with action queue and priorities.
  • Procedural map generation with seeded RNG for reproducibility.
  • AI behaviors that switch between aggressive, evasive, and support modes.
  • Combat effects: damage over time, shields, taunts, and interrupts.

Tech Stack

  • C++17, custom ECS, STL.
  • OpenGL for rendering, GLFW for windowing and input.
  • GLM for math, stb_image for assets.
  • CMake build, cross platform dev on macOS and Windows.

Architecture and Systems

Core Systems

  • PhysicsSystem - updates entity transforms and handles collisions.
  • CombatSystem - resolves turns, abilities, and status effects.
  • AISystem - pathfinding and behavior trees with threat evaluation.
  • RenderSystem - sprites, layers, and camera.
  • MapGen - seeded dungeon generation and placement rules.

Key Components

  • Transform, Velocity, Collider, Health, Mana.
  • Ability - cost, cooldown, range, and effect.
  • AIState - mode, target, and memory for last seen positions.

What I Built

  • Designed and implemented the ECS with efficient iteration over dense component arrays.
  • Created the turn engine with action queue, priorities, and interrupt rules.
  • Implemented pathfinding and behavior switching for enemies.
  • Built procedural level generation and spawn balancing tools.
  • Added debugging overlays: entity counts, frame time, and collision boxes.
